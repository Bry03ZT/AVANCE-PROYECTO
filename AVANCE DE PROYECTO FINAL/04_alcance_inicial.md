# **4\. Alcance inicial del proyecto**

El alcance del proyecto se delimita al diseño y desarrollo de un **prototipo funcional de un sistema web básico de gestión logística**, adaptado específicamente a la realidad operativa de un negocio pequeño como Mueblería J & M. Este sistema se centrará exclusivamente en la digitalización y centralización de tres procesos internos clave: el registro de **pedidos**, el control de **inventario** y la planificación de **entregas**.

El entregable principal será una interfaz web sencilla, intuitiva y accesible desde navegadores de teléfonos móviles o computadoras, que permita al vendedor, al encargado de almacén y al repartidor compartir la misma información en tiempo real, eliminando el uso de cuadernos y reduciendo la informalidad en la comunicación interna.

### **Limitaciones del proyecto**

Dado que el objetivo es lograr una transformación digital realista, aplicable y de rápida adopción (evitando complejidades tecnológicas innecesarias que dificulten su uso para el personal actual), el proyecto **excluye explícitamente** lo siguiente en esta fase inicial:

| Limitación (Fuera del alcance) | Justificación operativa |
| :--- | :--- |
| **Integración con pasarelas de pago o facturación electrónica** | El sistema funcionará de forma aislada para la gestión logística interna. La cobranza y facturación seguirán manejándose por los canales actuales para evitar altos costos de integración y licencias comerciales. |
| **E-commerce o catálogo virtual para el cliente** | El objetivo es solucionar los problemas internos ("arreglar la casa por dentro"). La venta seguirá dependiendo de la tienda física y las redes sociales. |
| **Optimización de rutas con Inteligencia Artificial o Google Maps** | Las entregas se organizarán en una lista digital estructurada (cliente, dirección, horario, producto). El repartidor seguirá decidiendo su ruta óptima basándose en su conocimiento local de Arequipa, sin algoritmos complejos. |
| **Integración oficial con la API de WhatsApp Business** | No se automatizará el envío masivo ni se usarán bots de WhatsApp oficiales para evitar los costos asociados a Meta. La coordinación con el cliente seguirá siendo manual pero apoyada en la información centralizada del nuevo sistema. |
